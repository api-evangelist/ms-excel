<p><span style="color: black;">We’re deprecating the <strong>previewImageUrl </strong></span><span style="color: black;">property in the</span> <a href="https://learn.microsoft.com/en-gb/graph/api/resources/insights-resourcevisualization?view=graph-rest-1.0">resourceVisualization</a> <span style="color: black;">complex type for <a href="https://nam06.safelinks.protection.outlook.com/?url=https%3A%2F%2Flearn.microsoft.com%2Fen-gb%2Fgraph%2Fapi%2Fresources%2Fofficegraphinsights%3Fview%3Dgraph-rest-1.0&amp;data=05%7C02%7Clauragra%40microsoft.com%7C61519c7c51e44cb5a9a508de058d563e%7C72f988bf86f141af91ab2d7cd011db47%7C1%7C0%7C638954301862759107%7CUnknown%7CTWFpbGZsb3d8eyJFbXB0eU1hcGkiOnRydWUsIlYiOiIwLjAuMDAwMCIsIlAiOiJXaW4zMiIsIkFOIjoiTWFpbCIsIldUIjoyfQ%3D%3D%7C0%7C%7C%7C&amp;sdata=pAYL%2F7uKyo6Nqk6pPmuMJsl%2BaRd9tkRR7z7Tn5TFiqU%3D&amp;reserved=0" title="https://learn.microsoft.com/en-gb/graph/api/resources/officegraphinsights?view=graph-rest-1.0">Insights items</a> where the referenced resource is a file attachment (</span><code><span style="font-size: 10.0pt; color: black;">resourceReference.type</span></code><span style="color: black;"> = </span><code><span style="font-size: 10.0pt; color: black;">fileAttachment</span></code><span style="color: black;">). This affects responses from the <a href="https://nam06.safelinks.protection.outlook.com/?url=https%3A%2F%2Flearn.microsoft.com%2Fen-us%2Fgraph%2Fapi%2Finsights-list-shared%3Fview%3Dgraph-rest-1.0%26tabs%3Dhttp&amp;data=05%7C02%7Clauragra%40microsoft.com%7C61519c7c51e44cb5a9a508de058d563e%7C72f988bf86f141af91ab2d7cd011db47%7C1%7C0%7C638954301862771161%7CUnknown%7CTWFpbGZsb3d8eyJFbXB0eU1hcGkiOnRydWUsIlYiOiIwLjAuMDAwMCIsIlAiOiJXaW4zMiIsIkFOIjoiTWFpbCIsIldUIjoyfQ%3D%3D%7C0%7C%7C%7C&amp;sdata=dZAHGSgAOdhJcMP5IvykVWli9qSsKDOrAf%2FegtAm43U%3D&amp;reserved=0" title="https://learn.microsoft.com/en-us/graph/api/insights-list-shared?view=graph-rest-1.0&amp;tabs=http">Insights List shared</a> and <a href="https://nam06.safelinks.protection.outlook.com/?url=https%3A%2F%2Flearn.microsoft.com%2Fen-us%2Fgraph%2Fapi%2Finsights-list-used%3Fview%3Dgraph-rest-1.0%26tabs%3Dhttp&amp;data=05%7C02%7Clauragra%40microsoft.com%7C61519c7c51e44cb5a9a508de058d563e%7C72f988bf86f141af91ab2d7cd011db47%7C1%7C0%7C638954301862783005%7CUnknown%7CTWFpbGZsb3d8eyJFbXB0eU1hcGkiOnRydWUsIlYiOiIwLjAuMDAwMCIsIlAiOiJXaW4zMiIsIkFOIjoiTWFpbCIsIldUIjoyfQ%3D%3D%7C0%7C%7C%7C&amp;sdata=hXSUHv0UOB%2FVUV%2FhhLGo2nVu%2FukLHvG68YmvJoxnZA0%3D&amp;reserved=0" title="https://learn.microsoft.com/en-us/graph/api/insights-list-used?view=graph-rest-1.0&amp;tabs=http">Insights List used</a> APIs when the underlying resource is an email attachment.</span></p>
<h2>What’s changing and why</h2>
<p>The <strong>previewImageUrl</strong> property is no longer returned for file attachment items. File attachments don’t have a consistent preview experience across Microsoft 365, so removing this property simplifies the API and aligns it with supported preview scenarios. Items that reference cloud files (like <strong>driveItem</strong>) aren&#8217;t impacted.</p>
<h3>Timeline</h3>
<ul>
<li><strong>Deprecation notice</strong>: Effective immediately</li>
<li><strong>Removal date</strong>: November 1, 2027</li>
</ul>
<h2><strong>Action required</strong></h2>
<ul>
<li>If your app uses <code>resourceVisualization.previewImageUrl</code> for file attachment insights, update your logic to handle the absence of this property.</li>
<li>For previews, prefer modern attachments (cloud files) and use the <a href="https://nam06.safelinks.protection.outlook.com/?url=https%3A%2F%2Flearn.microsoft.com%2Fen-gb%2Fgraph%2Fapi%2Fdriveitem-preview%3Fview%3Dgraph-rest-1.0&amp;data=05%7C02%7Clauragra%40microsoft.com%7C61519c7c51e44cb5a9a508de058d563e%7C72f988bf86f141af91ab2d7cd011db47%7C1%7C0%7C638954301862794175%7CUnknown%7CTWFpbGZsb3d8eyJFbXB0eU1hcGkiOnRydWUsIlYiOiIwLjAuMDAwMCIsIlAiOiJXaW4zMiIsIkFOIjoiTWFpbCIsIldUIjoyfQ%3D%3D%7C0%7C%7C%7C&amp;sdata=STGO7A7WOORgNePUSk%2F9bB6bEOHv0NZ%2B%2FiTeUIIdoPo%3D&amp;reserved=0">driveItem: preview</a> action where applicable.</li>
</ul>
